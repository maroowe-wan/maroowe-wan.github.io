# YouTube 교육 컨텐츠 자동 생성 파이프라인

이 프로젝트는 강좌명을 입력받아 YouTube 동영상 기반 교육 컨텐츠(HTML)를 자동 생성한다.

## 실행 방법

사용자가 "○○ 강좌 만들어줘" 라고 하면, 아래 단계를 순서대로 진행한다.
강좌명만 주어지면 즉시 1단계부터 시작한다.

## 파이프라인 단계 및 담당 에이전트

| 단계 | 작업 | 담당 에이전트 | 병렬 여부 |
|------|------|--------------|----------|
| 1 | 커리큘럼 설계 (한국어 spec, 최대 10강(필요에 따라 판단), 강당 15분 분량) | curriculum-designer | 단일 |
| 2 | YouTube 검색 (자막 有 + 평점 우수) | youtube-scout | 강좌별 병렬 |
| 3 | 자막 추출 | transcript-extractor | 강좌별 병렬 |
| 4 | 강의 내용 생성 **(영어 원본 → content.en.md)** | content-writer | 강좌별 병렬 |
| 4b | 다이어그램 보강 (영어 캡션, content.en.md) | diagram-architect | 강좌별 병렬 |
| 5 | 감수 + 보강 판단 **(영문 content.en.md 대상, 다이어그램·영어 표현 포함)** | content-reviewer (Gemini 기반 감수) | 단일 (수렴) |
| 6 | 실습/퀴즈 생성 **(영어 → exercise.en.json)** | exercise-builder | 강좌별 병렬 |
| 6b | 한국어 현지화 (content.md / exercise.json) + 커리큘럼 영문화 (01_curriculum.en.json) | content-localizer | 강좌별 병렬 |
| 7 | HTML 빌드 + GitHub Pages 사이트 구조화 (한/영) | html-publisher | 단일 (수렴) |

> **언어 정책 — 영어가 학습 본문의 원본(canonical).** 한·영 양쪽이 주요 서비스이며, 영어 원본 → 한국어 현지화가 총 품질 손실이 가장 적다(영어 작성 정밀도가 높고 EN→KO 열화가 KO→EN보다 견딜 만하다). 따라서 **본문·실습은 영어로 작성·감수**(4·4b·5·6)한 뒤 **6b에서 한국어로 현지화**한다. 단 **커리큘럼(설계 spec)은 한국어**(`01_curriculum.json`)이고 그 영문본은 6b가 만든다. 빌더는 파일명을 표시 언어로 다루므로(`content.md`=한국어 페이지, `content.en.md`=영어 페이지) 한국어 원본 시절의 레거시 강좌도 그대로 빌드된다.

## 핵심 라우팅 규칙

### 1. 출력 위치
- 각 강좌 작업물은 `courses/<강좌명_슬러그>/` 아래에 저장한다.
- 단계별 산출물:
  - `01_curriculum.json` — 커리큘럼 **한국어 spec** (1단계, curriculum-designer)
  - `lectures/<NN>_<제목>/sources.json` — 검색된 영상 목록 (2단계)
  - `lectures/<NN>_<제목>/transcript.txt` — 자막 (3단계)
  - `lectures/<NN>_<제목>/content.en.md` — 강의 내용 **영어 원본** (4단계, content-writer). **4b단계에서 diagram-architect가 같은 파일에 영어 Mermaid 다이어그램을 보강한다.**
  - `lectures/<NN>_<제목>/review.json` — 감수 결과 (5단계, 영문 본문 대상)
  - `lectures/<NN>_<제목>/exercise.en.json` — 실습/퀴즈 **영어 원본** (6단계, exercise-builder)
  - `lectures/<NN>_<제목>/content.md`, `lectures/<NN>_<제목>/exercise.json` — **한국어 현지화본**, `01_curriculum.en.json` — **커리큘럼 영문본** (6b단계, content-localizer)
- **최종 HTML은 강좌 폴더가 아니라 저장소 루트의 `docs/` 통합 사이트로 빌드한다** (GitHub Pages `/docs` 배포용, 7단계). 사이트는 **한국어(ko)·영어(en) 두 언어**로 빌드된다:
  - `docs/index.html` — **언어 라우터.** 브라우저 언어(`navigator.language`)·이전 선택(localStorage `eduLang`)에 따라 `docs/ko/` 또는 `docs/en/`로 자동 이동. JS 비활성 시 `<noscript>` 수동 링크 폴백.
  - `docs/<lang>/index.html` — 언어별 강좌 목록 랜딩 (`ko`, `en`)
  - `docs/<lang>/courses/<슬러그>/index.html` — 언어별 강좌 페이지. 각 페이지 상단에 수동 KO↔EN 토글.
  - `docs/assets/{css,js,img}/` — 모든 언어·강좌가 공유하는 공통 자산 (CSS/JS 인라인 금지, 외부 파일로 분리)
  - 공통 자산은 `없을 때만` 생성되어 손편집이 보존된다. 디자인/스크립트 변경을 반영하려면 `--reset-assets` 로 1회 재생성. 배포 절차는 `DEPLOY.md` 참조.
  - ⚠ **`docs/`는 빌드 산출물이다 — 손으로 직접 수정하지 말 것.** 본문·실습·번역의 모든 수정은 `courses/<슬러그>/lectures/...`의 원본 파일에만 하고 `scripts/build_html.py`로 재생성한다. (작업 CWD가 `docs/` 하위일 수 있으니 원본/산출물 경로를 혼동하지 않는다.)
  - 빌더는 `<base>.en.<ext>` 우선, 없으면 `<base>.<ext>`로 폴백한다. 신규 강좌는 영어가 원본(`content.en.md`)이고 한국어(`content.md`)는 6b 현지화본이며, 한국어 원본 시절의 레거시 강좌는 그 반대다 — 어느 쪽이든 파일명=표시 언어이므로 양 사이트 모두 정상 빌드된다.

### 1b. 강좌 슬러그·명명 규칙 (폴더명 ≠ 표시명)
- **폴더 슬러그는 URL-안전 ASCII**로만 만든다: **소문자·숫자·하이픈(`-`)만**, 공백·`&`·대문자·한글 금지. 빌더(`build_html.py`)가 **폴더명을 URL 경로·파일시스템 경로에 그대로** 쓰므로(슬러그 정제 없음), `&`나 대문자는 GitHub Pages·대소문자 무시 파일시스템에서 깨진다.
- **표시명은 `course_title`(`01_curriculum.json`)에 자유롭게** 둔다 — 특수문자·한글·예쁜 표기 가능(예: `K8S-PV&PVC`). 빌더가 HTML 이스케이프해 안전히 렌더링한다. **슬러그(폴더)와 표시명(course_title)은 분리**한다.
- **랜딩 목록은 슬러그 알파벳순** 정렬이다(별도 정렬 필드 없음). 순서를 통제하려면 슬러그에 **2자리 숫자 접두사**를 붙인다(예: `k8s-01-…` ~ `k8s-25-…` → 알파벳순=난이도순).

### 1c. 쿠버네티스(K8S) 시리즈 — 토픽별 단일 강의 강좌
- K8S 강좌는 **레벨 묶음(beginner/intermediate/…) 금지.** **토픽 1개 = 강좌 1개**로 만든다.
- 표시명 `course_title` = `K8S-{토픽}`(예: `K8S-Ingress`, `K8S-PV&PVC`), 폴더 슬러그 = `k8s-NN-토픽`(NN=난이도 순번, 예: `k8s-10-ingress`).
- 각 강좌는 **단일 강의**(`total_lectures: 1`, lecture `no: 1`)이며 `lectures/01_<제목>/` 하나만 둔다. **ko(`01_curriculum.json`)·en(`01_curriculum.en.json`) 커리큘럼을 모두** 만든다(en 없으면 영어 사이트에서 누락).
- **수정:** 원본 `courses/k8s-NN-*/lectures/01_*/`만 고치고 `python scripts/build_html.py --all`로 재빌드(`docs/`는 산출물, 직접 수정 금지). 영문 본문을 고치면 한국어 현지화본도 동기화한다(빌더의 `[STALE]` 경고 확인).
- **추가:** 난이도에 맞는 `NN`으로 `courses/k8s-NN-토픽/` 생성(중간 삽입이면 뒤 번호를 재정렬하거나 적절한 빈 번호 선택) → ko/en 커리큘럼 + 단일 강의 산출물(content/exercise/sources)을 채우고 재빌드.

### 2. 감수→보강 루프 (가장 중요)
5단계에서 content-reviewer가 **영문 본문(`content.en.md`)**을 대상으로 `verdict`를 판정한다(보강 루프는 영어에서만 돈다. 한국어 현지화는 영어 확정 후 6b에서 1회 수행):
- `PASS`: 다음 단계로.
- `REVISE`: content-writer를 **같은 강좌에 대해 다시 호출**해 영문 `content.en.md`를 고치되, review.json의 `issues` 와 `missing_topics` 를 입력으로 넘긴다. 이때 youtube-scout에게 추가 영상을 더 찾게 할 수도 있다.
- 재작성은 **최대 4회**까지만. 4회 후에도 REVISE면 `review.json`에 `escalate: true`로 기록하고(사람 검토용 내부 플래그이며 HTML에는 표시하지 않는다) 다음 단계로 넘어간다 (파이프라인을 멈추지 않는다).

### 3. 실습 형태 자동 판단 (6단계)
content-writer가 4단계에서 각 강의에 `lecture_type` 을 부여한다:
- `practical` (코딩/실습형): exercise-builder가 단계별 실습 과제 + 정답/해설 생성. **실습은 필수.**
- `theory` (이론형): 실습이 불가능하므로 **퀴즈로 대체** (객관식 3~5문항 + 해설).
- 판단 기준은 skills/의 가이드와 content-reviewer가 함께 검증한다.

### 4. 다이어그램 보강 (4b단계)
content-writer가 영문 본문을 쓴 뒤, diagram-architect가 같은 `content.en.md`를 읽어 **그림이 글보다 더 잘 전달하는 지점**에만 Mermaid 다이어그램(영어 캡션·라벨)을 삽입한다. 한국어 페이지의 다이어그램은 6b에서 캡션·라벨만 한국어로 옮겨 만든다.
- 대상: 구성도/아키텍처(flowchart), 상호작용 순서(sequenceDiagram), 유즈케이스, 분기 절차, 상태 전이, 클래스/관계 등.
- 단순 나열·정의처럼 텍스트로 충분한 곳에는 **넣지 않는다**(억지 그림 금지). 15분 강의당 0~3개가 적정.
- **다이어그램은 오직 이해를 높일 때만 둔다.** 본문·사실과 어긋나 오해를 유발할 소지가 있으면 사실에 맞게 고치고, **고쳐도 혼동이 남으면 삭제한다**(틀리거나 헷갈리는 그림은 없는 것만 못하다). 삭제 시 본문의 연결 문장과 영문본의 같은 블록도 함께 정리한다.
- 형식: ` ```mermaid <캡션> ` 펜스 블록(영문 원본은 영어 캡션, 한국어 현지화본은 한국어 캡션). 빌더(`build_html.py`)가 HTML에서 Mermaid.js로 렌더링하고, 오프라인이면 소스 텍스트 폴백으로 표시한다.
- 문법 규칙은 `skills/diagram-mermaid` 가이드를 따른다. 잘못된 문법은 그림 대신 코드가 노출되므로 5단계 감수에서 함께 검증한다.

### 5. 영상 품질 기준 (2단계)
- 자막(자동생성 포함, 단 한국어/영어 우선)이 **반드시 있는** 영상만.
- 좋아요 비율, 조회수, 채널 신뢰도, 최신성을 종합한 점수 상위만 수집.
- 강당 1~3개 영상을 확보 (15분 분량을 채우기 위해 복수 가능).
- 각 영상의 `subtitle_lang`(ko/en)을 **정확히 기록**한다. 빌더가 이 값으로 영문 페이지의 출처 영상을 필터링한다.

### 6. 한/영 이중 언어 (i18n)
사이트는 한국어·영어 두 버전을 만들고, 접속자 언어에 따라 자동 이동한다. **영어가 학습 본문의 원본**이다.
- **현지화(6b):** 영문 감수(5)가 끝난 강의에 대해 content-localizer가 **영문 원본을 한국어로 현지화**해 `content.md`·`exercise.json`을 만들고, **한국어 커리큘럼 spec을 영어로** 옮겨 `01_curriculum.en.json`을 만든다. 영문 본문이 확정된 뒤 현지화하므로 보강 루프는 영어만 돈다. 한국어 현지화본은 5단계 감수를 거치지 않으므로 content-localizer가 **번역투·자연스러움 점검**을 자체 품질 게이트로 수행한다. **영문 `content.en.md`를 나중에 수정하면 해당 `content.md`도 반드시 다시 현지화해 한/영 버전을 일치시킨다(동기화 필수). 빌더(`build_html.py`)가 mtime 기준으로 `[STALE]` 경고(한국어 현지화본이 영문 원본보다 오래됨)를 출력하니 빌드 로그에서 누락을 확인한다.**
- **검색은 언어 무관(2):** youtube-scout는 언어를 구분하지 않고 검색하되, 한국어·영어 자막 영상을 모두 보존하고 `subtitle_lang`을 기록한다. 영문 페이지가 비지 않도록 가능하면 영어 자막 영상을 최소 1개 포함한다.
- **출처 영상 노출 규칙(7, 빌더):** 한국어 페이지(`docs/ko`)는 **한+영 전체** 영상을, 영어 페이지(`docs/en`)는 **영어 자막 영상만**(`subtitle_lang`이 `en`으로 시작) 노출한다.
- **라우팅:** `docs/index.html`이 브라우저 언어/localStorage(`eduLang`)로 `ko`/`en`을 결정해 이동하고, 각 페이지의 KO↔EN 토글이 선택을 localStorage에 기억시킨다.

## 비용/성능 규칙
- 단순 검색·추출 단계는 빠른 모델, 생성 단계는 고품질 모델을 쓴다 (각 에이전트 frontmatter 참조).
- 감수(5)는 교차검증을 위해 **외부 모델(Gemini CLI)**에 위임한다. content-reviewer는 `scripts/review_with_gemini.py`를 호출하는 얇은 오케스트레이터(haiku)이며, 본문을 직접 수정하지 않고 결함을 review.json에 플래그만 한다.
- 강좌별 병렬 작업(2·3·4·4b·6·6b)은 강좌가 5개 이상일 때만 병렬화하고, 그 이하는 순차로 진행해 토큰 폭주를 막는다.
- 현지화(6b)는 생성이 아닌 변환이므로 경량 모델(sonnet)을 쓴다. 단 한국어 번역투 점검은 6b의 자체 품질 게이트다.

## 파일 산출·임시파일 규칙 (전 에이전트 공통)
- **저장소 루트(`edu-pipeline/`)에 파일을 만들지 않는다.** 모든 작업 산출물은 정해진 경로에만 둔다:
  - 강좌/강의 산출물 → `courses/<슬러그>/...`, `courses/<슬러그>/lectures/<NN>_*/...` (스크립트 호출 시 `--out`으로 이 경로를 준다)
  - 최종 사이트 → `docs/` (오직 `scripts/build_html.py`가 생성)
- **일회성 탐색·디버그용 임시 파일은 저장소 트리가 아니라 OS 임시 폴더**에 만든다(쉘 `$TEMP`/`%TEMP%`, Python `tempfile`). 쓰고 나면 지운다. `temp_*.json`, 루트 `*.py`, `temp/` 즉석 스크립트 금지.
- **일회성 산출물은 작업이 끝나면 반드시 삭제한다.** 다른 에이전트/사람과 공유하거나 재사용할 목적이 아닌, 그 자리 한 번의 작업을 위해 만든 스크립트·로그·중간 파일(예: 자막 일괄 병합용 즉석 스크립트, `extraction_log.txt` 같은 진행 로그)은 그 작업이 완료되는 즉시 지운다. 저장소에 남길지 애매하면 **남기지 않는 것이 기본값**이다 — 정식 파이프라인에 연결되거나 반복 재사용되는 도구만 `scripts/`에 보존하고, 그 경우 강좌명·경로를 하드코딩하지 말고 인자/상대경로로 일반화한다.
- 스크립트는 CWD=저장소 루트에서 돌므로 **상대경로 쓰기는 루트에 떨어진다** — 반드시 경로를 명시한다. 이 규칙은 PreToolUse(Write) 훅이 강제한다(루트 직속 경로 쓰기 차단). 단 훅은 Write 도구만 잡으므로, Bash/스크립트 쓰기는 이 지침으로 지킨다.
- 폴더·파일 탐색은 bash `find`/`ls` 대신 **Glob/Read** 도구를 쓴다(한글 경로 안전).

## 안전 규칙
- yt-dlp 등 외부 도구가 봇 차단(쿠키 요구)으로 실패하면, 해당 강좌를 건너뛰지 말고 사용자에게 알리고 대체 영상을 재검색한다.
- 저작권: 자막 원문을 그대로 복제해 컨텐츠로 쓰지 않는다. 반드시 재구성·요약·재작성한다. 출처(영상 URL)는 각 강의에 명시한다.
