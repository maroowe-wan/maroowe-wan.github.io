## Video 1: Easily Deploy Full Stack Node.js Apps on AWS EC2 (https://www.youtube.com/watch?v=nQdyiK7-VlQ)

In this video, we'll learn how to deploy a full stack Node.js application on AWS EC2.

First, set up an EC2 instance with the necessary runtime and dependencies. Install Node.js and npm on the EC2 instance.

Clone your application repository on the EC2 instance.

Install application dependencies using npm install.

Start your application using a process manager like PM2 which keeps the application running even after server restart.

Set up a reverse proxy like Nginx to forward requests to your Node.js application running on a specific port.

Configure Nginx to listen on port 80 and forward to your Node.js app on port 3000 or similar.

Use SSL certificate from Let's Encrypt for HTTPS support.

Configure the application to connect to your database. Use environment variables for database credentials.

Create a systemd service file to automatically start your application on server boot.

Use deployment scripts to automate the deployment process. Pull the latest code from repository and restart the application.

Set up a GitHub Actions workflow or use Jenkins to automate deployments when code is pushed.

The deployment workflow typically involves: pull code, install dependencies, build assets, restart application.

Use health checks to ensure the application is running properly after deployment.

Monitor application logs to debug issues. Use tools like CloudWatch for centralized logging.

## Video 2: 배포화 jenkins docker 1 #젠킨스 설치 (https://www.youtube.com/watch?v=mAPbPAtRPUw)

This video is part 1 of the Jenkins Docker deployment series, focusing on Jenkins installation.

Jenkins를 Docker 컨테이너로 실행하면 별도의 설치 프로세스가 필요하지 않습니다.

Docker를 먼저 설치하고, Jenkins Docker 이미지를 가져옵니다.

docker run 명령으로 Jenkins 컨테이너를 실행합니다.

Jenkins는 포트 8080에서 실행되므로 브라우저에서 localhost:8080으로 접근할 수 있습니다.

초기 비밀번호는 컨테이너 로그에서 확인할 수 있습니다.

Jenkins 웹 UI에 로그인하여 초기 설정을 완료합니다.

필요한 플러그인을 설치합니다.

관리자 계정을 생성합니다.

이제 Jenkins가 Docker 컨테이너 내에서 준비되었습니다.

## Video 3: 배포화 jenkins docker 3 #코드배포 #업데이트배포 (https://www.youtube.com/watch?v=j0_keQl-XAg)

This video is part 3 of the Jenkins Docker deployment series, covering code deployment and updates.

Jenkins 파이프라인을 구성하여 코드 배포를 자동화합니다.

Git 저장소에서 코드를 가져와 빌드합니다.

Docker 이미지를 빌드하고 푸시합니다.

EC2 인스턴스에 배포합니다.

배포 전에 기존 컨테이너를 중지하고 새로운 이미지를 풀합니다.

docker run으로 새 컨테이너를 시작합니다.

배포 후 헬스 체크를 수행하여 애플리케이션이 정상 작동하는지 확인합니다.

실패 시 이전 버전으로 롤백할 수 있도록 준비합니다.

배포 로그를 기록하여 나중에 추적할 수 있도록 합니다.

이렇게 자동화된 배포를 통해 수동 개입을 최소화하고 일관성을 유지합니다.
