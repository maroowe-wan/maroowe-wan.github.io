## Video 1: How to run Selenium TestNG tests in Jenkins (https://www.youtube.com/watch?v=ESQC5tH5Aow)

This video demonstrates how to set up testing in Jenkins. We'll show how to run Selenium TestNG tests as part of the CI pipeline.

Setting up the Maven integration with Jenkins allows automatic test execution. When you have unit tests or integration tests, they run automatically during the build process.

You can configure Jenkins to fail the build if tests don't pass. This is a critical part of CI/CD where quality gates prevent bad code from progressing.

The test stage in your pipeline should include running automated tests. These can be unit tests, integration tests, or UI tests.

In Jenkins, when tests fail, the pipeline stops and notifies the development team. This fast feedback loop is essential for maintaining code quality.

You can see test results in the Jenkins UI showing which tests passed and which failed. Test reports are automatically collected and displayed for each build.

The Maven test plugin integrates with Jenkins to execute tests and report results. You can specify test classes or test suites to run.

## Video 2: [SWTT] GITLAB CI CD로 간단한 자동배포 (https://www.youtube.com/watch?v=WyH_ihsIaik)

GitLab CI/CD를 사용한 자동배포에 대한 강의입니다. GitLab에서 CI/CD 파이프라인을 설정하여 자동으로 빌드, 테스트, 배포를 수행할 수 있습니다.

먼저 GitLab 저장소의 루트에 .gitlab-ci.yml 파일을 생성합니다. 이 파일에 CI/CD 파이프라인의 모든 단계를 정의합니다.

스테이지는 build, test, deploy 등으로 구성됩니다. 각 스테이지에는 여러 작업이 포함될 수 있습니다.

빌드 스테이지에서는 소스 코드를 컴파일하고 아티팩트를 생성합니다. 테스트 스테이지에서는 자동화된 테스트를 실행합니다.

배포 스테이지에서는 빌드된 애플리케이션을 서버에 배포합니다. 이는 수동 배포를 제거하고 자동화합니다.

GitLab Runner를 설정하여 CI/CD 작업을 실행합니다. Runner는 로컬 머신이나 원격 서버에서 작동할 수 있습니다.

환경 변수를 사용하여 빌드와 배포 과정을 구성합니다. 민감한 정보는 GitLab 보안 변수로 관리합니다.

성공적인 빌드 후 자동으로 배포되므로 수동 작업이 최소화됩니다.

## Video 3: Create Your First Pipeline in Jenkins (https://www.youtube.com/watch?v=wBvoerbHWEU)

This video covers creating a declarative pipeline in Jenkins. It shows how to structure a pipeline with stages and steps.

The pipeline starts with defining the agent. You can specify any agent or a specific label.

Stages are logical groupings of work. Common stages are checkout, build, test, and deploy.

Inside each stage, you define steps. Steps are individual commands or tasks.

Use the pipeline syntax tool in Jenkins to generate pipeline code. This makes it easier than writing from scratch.

For checking out code, use the git step with your repository URL and credentials.

For building Maven projects, use the withMaven step which sets up the Maven environment.

Run mvn clean package or mvn install to build and package the application.

Tests run automatically as part of the Maven lifecycle.

The pipeline fails if any stage fails, preventing bad code from progressing.

You can view the pipeline in the UI showing each stage and whether it passed or failed.
